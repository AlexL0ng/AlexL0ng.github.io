README.md的撰写方案

发表于 2022-04-01 更新于 2023-01-25 分类于 工具

为什么要撰写README.md？

用户体验

README.md文档就像一个产品说明书，我们做的每个项目就是一个产品，用户拿到手中的时候并不清楚如何使用，不了解其组成部分。就像是你买的第一包泡面，泡面包装袋上没有写配料，生产场地，烹调方法。那就只能靠用户自己摸索使用，这对用户体验来说无疑是非常糟糕。

自我梳理

编写README.md可以梳理自身对项目的理解，同时也锻炼自己的文档编写能力。

README.md的编写方案

参考自：Parabol

Overview

在这里，对你的项目进行简要的概述，附上项目的demo演示图，或者项目展示图也可以。

Stack Information

技术栈，在这里写下你的项目开发需要的哪些技术信息，类似：

Concern Solution
UI Framework React

Prerequisites

项目运行的先决必备条件

Installation

安装步骤，以及在本地将项目跑起来代码的执行步骤

Development
