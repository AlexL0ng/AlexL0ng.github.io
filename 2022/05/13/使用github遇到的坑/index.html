经过

于是乎，我带着大大的疑问去检查.gitignore文件，想着我添加的.idea/是不是无法识别到上面一层目录的文件。于是我又改成了**/.idea/；经过测试；还是没有效果。

经过一段时间的折磨，决定去谷歌一下。

很多博客说的原因是:

".gitignore文件只会ignore没有被staged的文件。也就是说，之所以.idea能push到remote远程仓库，是因为你已经执行了git add命令。"

说的情况确实和我情况十分相似，于是乎跟着博客我又进行了操作。

但是无论我怎么样照着博客运行指令，最后依然会在我git add .之后看见

此时的我产生了两个疑问。为什么我执行这两个命令之后，.idea文件又会被追踪呢？为什么.gitignore没有生效呢？

去git官方手册查询了一下，git add命令不仅是把文件存放到暂存区，同时也将对指定文件进行跟踪。

我的第一个问题得到了解决！

那么第二个疑问是如何得到解决的呢。

我在查看git add .的时候是这样描述的，将当前目录下的所有文件添加到暂存区。

由于项目进度只在bot文件夹里开发，我切换到了bot路径下运行了上面的那系列操作。

那些文件被成功忽略了。于是乎我有了个猜想，.gitignore文件缩在路径和你的终端路径要在同一目录下，才会执行。

我又切到项目的根目录进行验证，发现确实如此。然后我在根目录下copy了一份.gitignore文件，在根目录下进行验证。发现问题被完美的解决了。！！

又是学到就是赚到的一天！

总结

  • 对于项目初期的文件配置有了进一步了解,以及一些工作如果不在开始的时候做好，后期就会很麻烦！！
