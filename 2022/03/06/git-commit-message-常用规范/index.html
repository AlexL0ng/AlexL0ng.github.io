git commit message 常用规范

Q：为什么要使用git提交规范？

A：这样进行分类划分，更方便管理和调整。

git commit -m "type:xxxx"

git commit（指令）

-m（分支，这里指代master分支）

“type:xxxx”（用于描述本次提交内容；type：自定义规范，xxxx：描述内容）

type类型(每个团队各有不同)

以下是常用的用于标识提交的代码的完成了什么任务的标识类别

feat - 新功能 feature

fix - 修复 bug

docs - 文档注释

style - 代码格式(不影响代码运行的变动)

refactor - 重构、优化(既不增加新功能，也不是修复bug)

perf - 性能优化

test - 增加测试

chore - 构建过程或辅助工具的变动

revert - 回退

build - 打包
