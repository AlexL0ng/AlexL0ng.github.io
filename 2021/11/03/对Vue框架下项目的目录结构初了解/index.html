对Vue框架下项目的目录结构初了解

发表于 2021-11-03 更新于 2023-01-25 分类于 编程

Q:为什么要整理目录结构?

A:更快的了解Vue框架的结构组成部分，更好的使用Vue框架来开发项目。

dist 存放一些静态资源文件

nodel_modules 存放项目依赖文件，例如：引入echarts插件下载的文件

public 静态资源目录

src 存放Vue项目源代码

  • assets 存放资源（例如：图片）

  • commponents 存放公共组件

  • global（自定义） 存放接口相关文件

    • request（自定义） 存放接口请求相关文件
    • service（自定义） 存放调用接口的方法的文件

  • router（引入Vue Router） 存放配置路由的文件，引入Vue Router后会自动生成

  • views 存放Vue视图组件文件

  • App.vue 根组件，最后会被渲染到public/index.html中

  • main.js 入口文件，对插件工具的引用并创建，对Vue配置和挂载
    gitignore git管理工具用来忽略管理文件的文件，在里面可以添加本地开发不需要加载的一些文件名，本地打开项目就不会加载标记的文件

babel.config.js babel工具的配置文件

package.json 项目依赖的文件的信息

package-lock.json 记录项目依赖的文件的版本号

README.md 帮助文件，可以用来写一些项目的介绍和使用方法之类的

tests(自定义)存放测试代码
