读书笔记 -《Clean Code》- 序

发表于 2022-05-02 更新于 2023-01-25 分类于 学习

序

“对于软件而言，百分之八十或更多的工作量集中在我们美其名曰“维护”的事情上：其实就是修修补补。”

如果在开发的时候，只考虑把功能给实现，那确实会是很简单的。但是如果将“维护”考虑进去，考虑以后软件功能的可拓展性，减少代码之间依赖性等等。开发的细节，原则变得尤为重要。

“良好的软件实践遵循这些规程：专注、镇定和思考。”

专注与思考：最近和小波老师结对交流的时候；小波老师提出issues，我来解决issues，我总是会被issues以外知识诱惑。小波老师告诉我专注于issues来开发，可以使得我更加专注。

“整理（Seiri）[4]，或谓组织（想想英语中的sort（分类、排序）一词）。搞清楚事物之所在——通过恰当地命名之类的手段——至关重要。觉得命名标识无关紧要？读读后面的章节吧。 整顿（Seiton），或谓整齐（想想英文中的systematize（系统化）一词）。有句美国老话说：物皆有其位，而后物尽归其位（A place for everything, and everything in its place）。每段代码都该在你希望它所在的地方——如果不在那里，就需要重构了。
清楚（Seiso），或谓清洁（想想英文中的shine（锃亮）一词）。清理工作地的拉线、油污和边角废料。对于那种四处遗弃的带注释的代码及反映过往或期望的无注释代码，本书作者怎么说的来着？除之而后快。
清洁（Seiketsu），或谓标准化。有关如何保持工作地清洁的组内共识。本书有没有提到在开发组内使用一贯的代码风格和实践手段？这些标准从哪里来？读读看。
身美（Shitsuke）[5]，或谓纪律（自律）。在实践中贯彻规程，并时时体现于个人工作上，而且要乐于改进。”

对于5S标准思考：

整理(Seiri)或者说是sort(分类、排序) :

以对象，属性，函数之类的分类，排序吗？

整顿(Seiton)或者说是整齐(systematize系统化) :

我的理解是分类，归档。

清楚(Seiso)或者说是shine(锃亮) :

冗余，无用的注释要除之而后快。不够整洁，不够清楚，重复逻辑的代码需要整理。

清洁(Seiketsu)或者说是标准化 :遵循标准化，团队内部协作，组内需要约定好代码风格；这样便于团队的开发效率，以及代码的整洁性。个人写代码的时候，也要遵守”规范”，不然一个项目在这个模块是这个风格，另一个模块又是另外一个风格，会造成代码看起来乱七八糟的。

身美(Shitsuke)或者说是纪律(自律) :

生活和工作，贯彻自己订下的规程计划，常常改进。

思考

读完序之后，感受到的是作者向我描述了从事开发这一行业的现状；这种问题是共通的，作者从其他行业的解决问题的经验中，汲取我们这一行业所需要的养分。

为什么要撰写README.md？

用户体验

README.md文档就像一个产品说明书，我们做的每个项目就是一个产品，用户拿到手中的时候并不清楚如何使用，不了解其组成部分。就像是你买的第一包泡面，泡面包装袋上没有写配料，生产场地，烹调方法。那就只能靠用户自己摸索使用，这对用户体验来说无疑是非常糟糕。

自我梳理

编写README.md可以梳理自身对项目的理解，同时也锻炼自己的文档编写能力。

家常菜之蛋炒饭

发表于 2022-03-20 更新于 2023-01-25 分类于 生活

一、食材准备

1、准备鸡蛋5个，隔夜米饭，姜100g，蒜100g，葱适量，二荆条青椒200g，辣椒面少许，榨菜一小袋

PS：可以自行DIY加入食材

二、食材处理

1、将材料都洗干净后；姜葱蒜榨菜切成粒装，二荆条青椒切小柱状。

2、鸡蛋打好，搅拌均匀。

3、米饭用饭勺尽量弄散开，方便等下炒饭。

三、开始烹饪

1、大火热锅倒油，油适量，待油起烟；倒入鸡蛋，一边用铲弄散一边翻炒；待鸡蛋都炒凝固。

2、继续大火翻炒鸡蛋，然后加入姜蒜青椒翻炒；待锅中炒的爆香后加入一部分葱，继续翻炒30s。

3、此时，倒入米饭后，用锅铲翻炒弄散；将米饭和其他食材充分混合后且米饭颗粒分明后，关小火继续翻炒。

4、倒入适量的盐和辣椒面，继续小火翻炒均匀后调味。待锅气起来后，加入剩余的葱，适量味精翻炒均匀后出锅。

这样一份色香味俱全的蛋炒饭就完成了。

家常菜之花椰菜抄肉片

发表于 2022-03-16 更新于 2023-01-25 分类于 生活

一、食材准备

1、准备好500g猪后腿肉，500g花椰菜，200g二荆条辣椒，1个鸡蛋，适量花椒，适量蒜瓣，适量豆瓣酱。

二、食材处理

1、将材料都洗干净后；猪肉切片，花椰菜切成小份；二荆条辣椒滚刀切，蒜瓣拍一下切块。

2、将鸡蛋打入猪肉里面，放入适量的芡粉，用手抓匀。

3、炒锅放水加热；水沸腾之后，将花椰菜放入锅内（花椰菜要全部没入水中），大约2分钟后，捞起花椰菜沥干；将水倒掉。

三、开始烹饪

1、大火热锅倒油，油可以适量多点。当油刚开始冒烟的时候，转中火，然后将肉倒入锅中翻炒；翻炒至肉片变色（时间过长的话，肉会变老）。将肉盛起来。

2、此时，如果锅内的油过少的话；添加少许油，大火烧热后；放入适量豆瓣酱，炒香后；再放入蒜瓣和花椒，10秒后放入二荆条辣椒翻炒30秒左右后；再将花椰菜放入锅中翻炒，锅内的花椰菜染上豆瓣色之后；再倒入肉片翻炒染色，大约3分钟后；试味道后，自行添加盐。再次翻炒均匀后，再次试味道，确定味道后。关小火，放味精少许，翻炒均匀起锅。

这样一份家常的花椰菜就完成了，十分下饭!!

git commit message 常用规范

发表于 2022-03-06 更新于 2023-01-25 分类于 工具

git commit message 常用规范

Q：为什么要使用git提交规范？

A：这样进行分类划分，更方便管理和调整。

git commit -m "type:xxxx"

git commit（指令）

-m（分支，这里指代master分支）

“type:xxxx”（用于描述本次提交内容；type：自定义规范，xxxx：描述内容）

type类型(每个团队各有不同)

以下是常用的用于标识提交的代码的完成了什么任务的标识类别

feat - 新功能 feature

fix - 修复 bug

docs - 文档注释

style - 代码格式(不影响代码运行的变动)

refactor - 重构、优化(既不增加新功能，也不是修复bug)

perf - 性能优化

test - 增加测试

chore - 构建过程或辅助工具的变动

revert - 回退

build - 打包
